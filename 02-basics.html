Part 2

기본 사용법

4가지 모드, 핵심 슬래시 명령어, 단축키를 익히고 첫 실습을 해봅니다.

⏱ 30분

첫 실행

1

작업 폴더 만들기

바탕화면에 my-workshop 폴더를 만들고 VS Code로 열기. (File → Open Folder)

2

터미널에서 claude 실행

VS Code 하단 터미널 열기 (Ctrl+`) → claude 입력 → 테마 선택 (Light 권장)

3

첫 대화

안녕하세요! 이 폴더에 "hello.txt" 파일을 만들어줘. 내용은 "Claude Code 워크샵 시작!" 입력 후 Enter

💡 터미널 = 대화창
터미널이 무섭게 느껴질 수 있지만, 그냥 Claude와 대화하는 창이라고 생각하세요. 명령어를 외울 필요가 없습니다. 하고 싶은 걸 말로 입력하면 됩니다.

4가지 Permission Mode

Shift+Tab을 누를 때마다 모드가 순환됩니다. 모드는 오른쪽 하단에 표시됩니다.

모드 파일 수정 명령어 실행 언제 쓸까?
Default 매번 확인 매번 확인 ✅ 처음 시작할 때 (권장)
Auto-accept Edits 자동 승인 매번 확인 어느 정도 익숙해지면
Plan Mode 읽기 전용 불가 작업 전 계획 검토할 때
⚠️ Dangerous Skip 자동 승인 자동 승인 Git 완비 후 + 경험 충분할 때만
⚠️ Dangerous Skip Permission 주의
이 모드는 Claude가 마음만 먹으면 내 컴퓨터의 모든 파일을 삭제할 수 있는 권한을 줍니다. Default 모드와 Plan 모드로 10번 이상 작업을 잘 컨트롤할 자신이 생긴 후, 반드시 Git 안전망을 구축한 다음에 사용하세요.

Plan Mode 실습

Plan Mode는 Claude가 파일을 건드리지 않고 계획만 세웁니다. 큰 작업을 시작하기 전에 항상 Plan Mode로 먼저 확인하세요.

Claude Code 입력
내 폴더에 간단한 메모 앱 HTML 파일을 만들려고 해.
어떤 파일들이 필요하고 어떻게 만들 건지 계획 세워줘. 지금은 만들지 말고.

→ 계획을 보고 방향이 맞으면 Shift+Tab으로 Default 모드로 전환해서 실행!

핵심 슬래시 명령어

프롬프트 입력창에서 /를 입력하면 목록이 나옵니다.

명령어역할언제?
/init 현재 폴더를 분석해서 CLAUDE.md 자동 생성 새 프로젝트 시작 시
/plan Plan Mode 진입 (읽기 전용 분석) 큰 작업 전 검토
/clear 대화 기록 초기화 (새 세션 시작) 하나의 작업 완료 후
/compact 긴 대화를 압축해서 컨텍스트 절약 대화가 길어질 때
/resume 이전 세션 불러오기 /clear 후 돌아가고 싶을 때
/cost 현재 세션 API 비용 확인 비용 모니터링
/help 모든 명령어 목록 보기 언제든지
📊 컨텍스트 관리 원칙
AI 모델은 대화를 "기억"하는 게 아니라 모든 대화 내용을 매번 다시 읽습니다. 대화가 길어질수록 느려지고 비용이 늘어납니다.

Anthropic 엔지니어들의 권장 패턴:
• 질문-답변 5쌍 이내에 한 작업 단위 완료
• 작업 완료 → /clear로 새 세션 시작
• 컨텍스트 50% 남았을 때 → /compact으로 미리 압축

키보드 단축키

Shift+Tab 모드 전환
Shift+Enter 줄바꿈 (Enter는 전송)
Ctrl+C 작업 중단
Esc 현재 입력 취소
EscEsc 직전 작업 취소 (되돌리기)
!명령어 터미널 Bash 직접 실행
⚠️ Esc × 2 주의사항
Esc를 두 번 누르면 Claude가 수정한 파일을 되돌립니다. 단, 사용자가 직접 수정한 내용은 되돌아가지 않습니다. Claude 작업만 취소됩니다.

! 로 터미널 명령어 실행

Claude Code 입력창에서 !로 시작하면 터미널 명령어를 직접 실행합니다. 에러가 나면 그 내용을 자동으로 컨텍스트로 넘겨줍니다.

예시
!ls           # 현재 폴더 파일 목록
!node app.js  # 파일 실행
!git status   # Git 상태 확인

실습 과제

🎯 실습 1 — 파일 생성 체험
Default 모드에서 아래를 입력하고, 승인 화면에서 y/n 선택을 체험해보세요.

"오늘 해야 할 일 3가지를 담은 todo.md 파일을 만들어줘"
🎯 실습 2 — Plan Mode 체험
Shift+Tab으로 Plan Mode로 전환 후:

"todo.md에 내일 할 일 3개를 추가하고 싶어. 어떻게 수정하면 좋을지 제안해줘"

계획을 확인 후 마음에 들면 다시 Shift+Tab으로 Default 모드로 전환해서 실행!
🎯 실습 3 — 컨텍스트 관리
위 실습이 끝나면 /cost로 비용 확인 → /clear로 새 세션 시작을 해보세요.
