바이브코딩 5단계 사이클

① 기획
→
② 플랜
→
③ 액션+저장
→
④ 확인

⑤ 수정+반복
① 기획GitHub 레포 생성 → Claude에게 기획 문서 초안 요청 → 직접 읽고 확인
② 플랜Plan Mode에서 페이즈별 작업 계획 MD 파일 생성 → 방향 확인
③ 액션+저장코드 작성 → git commit push → /clear → /init → 다시 저장
④ 확인MCP(Chrome)으로 결과 테스트 → 에러 분석 보고
⑤ 수정+반복문제 → 작은 플랜 → 수정 / 3번 실패시 세이브 지점으로 복귀

Git 핵심 명령어

명령어역할
git clone [URL]GitHub → 내 컴퓨터
git add .변경 파일 전체 선택
git commit -m "메시지"저장 (Claude가 자동 작성)
git pushGitHub에 업로드
git pullGitHub → 내 컴퓨터

💡 초보자는 commit + push + pull 3개만으로 충분!
Claude에게 "현재 상태 저장해줘"라고 하면 자동 실행.

효과적인 프롬프팅 원칙

  • 구체적으로 — "B열=서울 행만 필터링해서 새 파일로"
  • 검증 기준 포함 — 예상 결과나 샘플 데이터 함께 제공
  • Plan Mode 먼저 — 큰 작업 전에 계획 확인 필수
  • @파일명 참조 — 관련 파일을 직접 컨텍스트로
  • 반복 개선 OK — 5~10번 수정이 정상 과정
  • 막히면 /clear — 같은 실수 2번 반복 시 초기화

컨텍스트 관리

  • 질문-답변 5쌍 이내에 작업 완료 → /clear
  • 컨텍스트 50% 남았을 때 → /compact 미리 실행
  • 작업 완료 → /init → git push → /clear
  • /clear 후 돌아가고 싶으면 → /resume

⚠️ Anthropic 엔지니어들도 평균 5쌍 이내에 세션 종료
