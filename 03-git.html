Part 3

Git 안전망

바이브코딩에서 Git은 선택이 아닌 필수입니다. 코드의 타임머신 — 언제든 되돌릴 수 있는 안전망을 만드세요.

⏱ 30분

왜 Git이 필요한가?

문서 작업할 때 이런 파일들을 만든 적 없으신가요?

여러분의 폴더
보고서_최종.docx
보고서_최최종.docx
보고서_진짜최종.docx
보고서_진짜진짜최종_수정본.docx
보고서_최종_230614.docx

Git은 이 문제를 자동으로 해결합니다. 모든 변경 내역을 자동으로 기록하고, 언제든 과거 어느 시점으로 돌아갈 수 있습니다.

바이브코딩에서는 특히 중요합니다. AI가 만든 코드를 Dangerous 모드로 실행하다가 뭔가 잘못되면? Git이 있으면 한 방에 복구.

🎮 게임 비유
Git은 게임의 세이브 포인트와 같습니다. 보스 몬스터 잡기 직전에 저장하듯, 큰 작업 시작 전에 "현재 상태 저장해줘"라고 Claude에게 명령하면 됩니다.

Git vs GitHub

내 컴퓨터 파일들이 여기 있음
⇄
Git 로컬 버전 관리 도구
⇄
GitHub 클라우드 저장소
"코드용 구글 드라이브"
  • Git — 내 컴퓨터에서 변경 내역을 기록하는 도구
  • GitHub — Git 내역을 클라우드에 올려서 다른 기기에서 접근하거나 팀과 공유

핵심 명령어 7가지

이걸 직접 외울 필요는 없습니다. Claude Code가 대신 입력해줍니다. 개념만 이해하면 충분해요.

git clone [URL] GitHub에 있는 프로젝트를 내 컴퓨터로 통째로 가져오기
git add [파일] 저장할 파일을 선택 (무대 위로 올리기)
git commit -m "메시지" 선택한 파일 저장 + 어떤 작업인지 메모 남기기 (Claude가 메시지 자동 작성)
git push 내 컴퓨터 저장 내역 → GitHub에 업로드
git pull GitHub 최신 버전 → 내 컴퓨터로 받아오기
git branch [이름] 현재 상태를 유지하며 새 시간선 만들기 (실험용 작업 공간)
Pull Request (PR) 브랜치에서 작업한 내용을 메인에 합치도록 요청
✅ 초보자 전략
혼자 작업할 때는 commit + push + pull 3개만 잘 써도 충분합니다. 브랜치와 PR은 팀 협업에서 더 중요합니다.

Claude Code와 함께 Git 사용하기

예전에는 Git 명령어를 직접 입력해야 했지만, 이제는 Claude에게 말로 시키면 됩니다.

이렇게 시키면 됩니다
현재 상태 저장해줘                    → git add + commit + push 자동 실행
오늘 작업 내용 커밋해줘                → commit 메시지도 Claude가 작성
GitHub에 올려줘                       → push 자동 실행
아까 작업하기 전 상태로 돌아가줘        → git reset 또는 checkout
🤖 커밋 메시지 자동 생성
Claude Code는 작업 내용을 파악해서 의미 있는 커밋 메시지를 자동으로 작성해줍니다. 나중에 히스토리를 보면 무슨 작업인지 쉽게 이해할 수 있어요.

레포지토리 관리 원칙

  • 프로젝트마다 레포지토리 따로! — 너구리 게임이랑 회사 대시보드를 같은 레포에 넣으면 안 됩니다
  • VS Code에서 루트 디렉토리는 반드시 해당 프로젝트 폴더만 열기
  • 프론트엔드/백엔드는 규모에 따라 같이 또는 따로 관리
