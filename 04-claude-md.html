Part 4

CLAUDE.md 설정

Claude에게 이 프로젝트에서 지켜야 할 행동 지침을 알려주는 파일. 한 번 설정하면 매번 설명할 필요가 없어집니다.

⏱ 30분

CLAUDE.md란?

CLAUDE.md는 Claude에게 주는 온보딩 문서입니다.

새 인턴이 입사하면 회사 규정, 프로젝트 현황, 업무 방식을 알려주잖아요. CLAUDE.md가 그 역할을 합니다. Claude는 세션을 시작할 때마다 이 파일을 자동으로 읽습니다.

🔄 자동 로딩
claude를 실행할 때마다 Claude는 자동으로 CLAUDE.md를 읽습니다. 그래서 매번 같은 설명을 반복할 필요가 없어집니다.
"항상 한국어로 답해줘", "파일 수정 전에 꼭 확인해줘" 같은 걸 한 번만 써두면 됩니다.

파일 위치 계층

여러 위치에 CLAUDE.md를 둘 수 있으며, 더 구체적인 것이 우선순위가 높습니다.

1
전역 설정 — ~/.claude/CLAUDE.md
모든 프로젝트에 공통으로 적용되는 개인 설정. "항상 한국어로 답변", "코드 설명 자세하게" 같은 개인 취향.
2
프로젝트 설정 — ./CLAUDE.md
현재 프로젝트에만 적용. Git으로 팀과 공유. 프로젝트 구조, 기술 스택, 코딩 규칙 등.
3
개인 로컬 — ./CLAUDE.local.md
.gitignore에 포함되어 팀에게 공유되지 않는 개인 설정.

/init 으로 자동 생성하기

직접 처음부터 작성하기 어렵죠. /init을 실행하면 Claude가 현재 프로젝트를 분석해서 초안을 자동으로 작성해줍니다.

Claude Code 입력
/init

실행 후 생성된 CLAUDE.md를 VS Code에서 열어서 꼭 한 번 읽어보세요. Claude가 이 파일을 기반으로 모든 작업을 진행하기 때문입니다.

⚠️ 반드시 읽어보세요
Claude Code가 이 파일로 항상 작업을 진행합니다. AI가 무슨 방향으로 일하는지 알고, 발생한 실수의 원인을 파악하려면 CLAUDE.md를 최소 한 번은 읽어봐야 합니다.

CLAUDE.md에 넣으면 좋은 것들

CLAUDE.md 예시
# 프로젝트 개요
이 프로젝트는 팀 주간 보고서 자동화 도구입니다.
Google Sheets에서 데이터를 읽어 Slack으로 요약을 보냅니다.

# 기술 스택
- Python 3.11
- Google Sheets API (MCP)
- Slack API (MCP)

# 코딩 규칙
- 모든 답변은 한국어로
- 파일 수정 전에 반드시 사용자에게 확인 요청
- 함수명은 snake_case 사용
- 에러 메시지도 한국어로 작성

# 주의 사항
- Google API 키는 .env 파일에만 저장 (절대 코드에 하드코딩 금지)
- config.json은 수정하지 말 것

효과적인 CLAUDE.md 작성 원칙

  • 구체적으로 — "잘 만들어줘" ❌ vs "camelCase 파일명 사용, 주석은 한국어로" ✅
  • 200줄 이내 — 너무 길면 컨텍스트 낭비
  • 금지사항 명시 — "~하지 말 것" 형태가 효과적
  • 자주 업데이트 — 새 규칙이 생기면 즉시 추가

init 실행 타이밍

언제 /init을 실행하면 좋을까요?

  • 새 프로젝트 시작할 때 — 폴더 구조와 기술 스택을 문서화
  • 큰 작업이 완료된 후 — 새로운 파일과 구조 변경사항을 반영
  • /clear 후 새 세션 시작할 때 — 이전 맥락을 CLAUDE.md에 녹여두면 매번 설명 불필요
💡 패턴: init → commit → clear
작업이 끝나면: /init 실행 → CLAUDE.md 업데이트 → git commit push → /clear
다음 세션에서 Claude는 CLAUDE.md를 읽어서 이전 맥락을 이해하고 바로 작업을 이어갑니다.

Auto-memory (자동 기억)

CLAUDE.md 외에도 Claude는 작업하면서 중요한 내용을 자동으로 기억합니다.

Claude Code 입력
/memory

Claude가 자동으로 저장한 기억 목록을 볼 수 있습니다. 잘못된 기억이 있으면 수정하거나 삭제할 수 있어요.

실습

🎯 실습 — CLAUDE.md 만들고 규칙 추가하기
  1. /init 실행 → 생성된 CLAUDE.md를 VS Code에서 열기
  2. 파일 내용 읽어보기
  3. 맨 아래에 "내 규칙" 섹션 추가:
    ## 내 규칙
    - 항상 한국어로 답변
    - 파일 수정 전에 반드시 확인 요청
  4. 저장 후 /clear → 다시 claude 실행 → CLAUDE.md 내용이 반영되었는지 확인 ("뭐 알고 있어?" 라고 물어보세요)
✅ 이 단계 완료 체크
